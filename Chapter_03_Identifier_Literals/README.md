# Chapter 03: Identifiers, Literals, and Comments

Welcome to Chapter 3 of the JavaScript Concepts module! This folder covers the foundational rules for naming variables and leaving notes in your JavaScript code.

## 📂 Topics Covered in this Chapter

### 1. JavaScript Identifiers & Naming Rules
In JavaScript, an **identifier** is a name given to variables, functions, classes, etc. 
* **Files to review:** `06_Identifier_Rules.js`, `JS_Identifier_Rules.js`

**Key Rules to Remember:**
- Names **must** begin with a letter (a-z, A-Z), an underscore (`_`), or a dollar sign (`$`).
- Subsequent characters can be letters, numbers, underscores, or dollar signs.
- Identifiers are **case-sensitive** (`Score` and `score` are different variables).
- You **cannot** use JavaScript reserved keywords (like `var`, `function`, `return`) as identifiers.
- Hyphens (`-`) are not allowed in variable names (they are treated as subtraction operators).

### 2. Naming Conventions (Casing)
Different programming teams use different styles to make their code readable.
* **Files to review:** `07_Identifier_Part2.js`

**Common Conventions:**
- **`camelCase`**: First word is lowercase, following words are capitalized. *(Standard for JS variables/functions)*
- **`snake_case`**: Words separated by underscores.
- **`PascalCase`**: Every word starts with a capital letter. *(Standard for JS Classes/Components)*
- **`MACRO_CASE`**: All uppercase with underscores. *(Standard for fixed constants)*

### 3. JavaScript Comments
Comments are text notes added to the code to explain what the code is doing. The JavaScript engine ignores comments when running the code.
* **Files to review:** `08_Comments.js`

**Types of comments:**
- **Single-line comments**: Started with `//`
- **Multi-line comments**: Enclosed between `/*` and `*/`
